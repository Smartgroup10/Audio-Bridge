Audio Bridge

Plataforma de voz con IA para automatizar la atencion telefonica de notarias. Conversacion natural en espanol con transcripcion en tiempo real.

Go + OpenAI Realtime + Asterisk AudioSocket + Vue 3

Que es Audio Bridge

Audio Bridge es un sistema que intercepta llamadas telefonicas entrantes a notarias y las conecta con una IA conversacional (OpenAI Realtime API) capaz de hablar en espanol en tiempo real. La IA puede resolver consultas, dar informacion sobre servicios notariales y transferir la llamada a la extension correcta cuando sea necesario.

El sistema es multi-tenant: cada notaria tiene su propia configuracion (prompt, horarios, destinos de transferencia) y comparten la misma infraestructura.

Para el llamante

Llama al telefono de la notaria y habla con una recepcionista virtual que entiende espanol, responde preguntas y transfiere cuando es necesario.

Para la notaria

Panel de administracion con llamadas activas, transcripcion en vivo tipo chat, historial completo, grabaciones y configuracion del asistente.

Flujo de una llamada

1
PSTN -> PekePBX
Llamada entra al DDI de la notaria. Asterisk ejecuta el dialplan custom del tenant.
2
Dialplan -> Bridge (precreate)
curl POST al bridge con tenant_id, caller_id y ddi. El bridge evalua horario, concurrencia y reglas.
3
Bridge devuelve UUID
Si la IA debe atender, devuelve un UUID de sesion. Si no, el dialplan sigue el flujo normal de PekePBX.
4
Asterisk -> AudioSocket
Conecta audio bidireccional TCP al bridge usando el UUID. PCM16 signed little-endian, 8kHz mono.
5
Bridge -> OpenAI Realtime
Abre WebSocket a la API Realtime de OpenAI con el prompt personalizado del tenant.
6
Conversacion en vivo
Audio fluye: Caller <-> AudioSocket <-> Bridge <-> OpenAI WebSocket. Cada utterance se transcribe y se emite por SSE.
7
Transfer o resolucion
Si la IA detecta que debe transferir, llama una function tool. El bridge cierra AudioSocket y Asterisk continua el dialplan hacia el destino. Si la consulta se resuelve, cuelga directamente.
8
Persistencia
La llamada se guarda en SQLite con transcripciones completas, grabaciones WAV separadas (caller/IA), duracion y motivo de fin.

Multi-Tenant

Cada notaria es un tenant independiente con su propia configuracion:

Prompt personalizado

Contexto unico de la notaria: nombre, servicios, horarios, forma de hablar.

Transfer targets

Extensiones y numeros a los que la IA puede transferir (departamentos, notario, etc).

DDIs asignados

Numeros de telefono publicos que activan la IA para ese tenant.

Limites y horarios

Max llamadas concurrentes con IA, horario de atencion, comportamiento fuera de horario.

Integracion con PekePBX

PekePBX soporta contextos custom por empresa y por DDI. Audio Bridge se integra con un fichero de dialplan por tenant:

; Fichero: /etc/asterisk/custom/20186.conf
; Notaria Jose Cela - tenant 20186

[sub_custom_20186_pstn_in]
exten => _X.,1,Verbose(Audio Bridge: ${CALLER} -> ${CALLEE} company ${DST_COMPANY})

; 1. Preguntar al bridge si debe ir a IA
same => n,Set(RESULT=${SHELL(curl -s -X POST http://localhost:8080/api/v1/calls/precreate ...)})
same => n,Set(CALL_UUID=...)

; 2. Si no hay UUID -> flujo normal PekePBX
same => n,GotoIf($["${CALL_UUID}" = ""]?noai)

; 3. Conectar AudioSocket al bridge
same => n,Answer()
same => n,AudioSocket(${CALL_UUID},127.0.0.1:9092)

; 4. Post-AudioSocket: si hay transfer
same => n,GotoIf($["${TRANSFER_TARGET}" != ""]?transfer)
same => n,Hangup()

same => n(transfer),Dial(SIP/pd${DST_COMPANY}/${TRANSFER_TARGET},60)
same => n,Hangup()

same => n(noai),Return()

Opciones de contexto disponibles en PekePBX:

ContextoAlcanceUso
[sub_custom_general_pstn_in]Todas las empresasReglas globales
[sub_custom_{TENANT}_pstn_in]Todas las llamadas de un tenantIA para toda la notaria
[sub_custom_{TENANT}_{DDI}]Un DDI especificoIA solo en ciertos numeros

Despliegue

VPS

Binario Go compilado, ejecutado como servicio systemd. SQLite local. Grabaciones en disco. Panel Vue pre-built como static files.

PekePBX

Ficheros .conf en /etc/asterisk/custom/. Un fichero por tenant. dialplan reload para aplicar cambios.

# Compilar y desplegar
cd /opt/audio-bridge
go build -o audio-bridge ./cmd/bridge/
systemctl restart audio-bridge

# Frontend
cd web && npm run build
# dist/ se sirve automaticamente desde /admin/
