Introducción

¿Qué es Audio Bridge?

Un componente de software que actúa como intermediario entre la centralita telefónica de las notarías y el módulo de Inteligencia Artificial del CTN, permitiendo que las llamadas sean atendidas de forma automatizada.

Captura de llamadas

Recibe el audio de las llamadas entrantes y salientes desde la centralita Asterisk de la notaría mediante protocolo AudioSocket (TCP).

Conexión con la IA

Reenvía el audio en tiempo real al módulo de IA del CTN mediante WebSocket seguro (WSS) y retransmite las respuestas de voz al llamante.

Gestión de acciones

Ejecuta transferencias a extensiones humanas, origina llamadas salientes, graba el audio y registra las transcripciones de cada conversación.

Concepto clave: Audio Bridge no es una centralita ni un sistema de IA. Es la pieza que conecta ambos mundos, traduciendo protocolos de audio y gestionando las acciones que la IA solicita sobre la telefónía.

Escenarios de despliegue

¿Dónde encaja Audio Bridge?

El sistema se adapta a tres modelos de implantación según la infraestructura de cada notaría

A

Complemento a la telefonía existente

Audio Bridge se despliega junto a la centralita actual de la notaría. Las llamadas se desvían al Bridge únicamente cuando deben ser atendidas por la IA. El resto del flujo telefónico permanece inalterado.

Flujo operativo

Ciclo de vida de una llamada

Secuencia completa desde que un ciudadano llama hasta que la conversación finaliza

1. Llamada entrante

El ciudadano llama al número de la notaría. La centralita Asterisk recibe la llamada y la enruta al Audio Bridge mediante AudioSocket.

2. Conexión con la IA

El Bridge abre una sesión WebSocket segura con el módulo de IA, transmitiendo el contexto: notaría, número del llamante, horario, DDI de origen.

3. Conversación en tiempo real

El audio del llamante se reenvía a la IA (STT → LLM → TTS). Las respuestas se reproducen al llamante. El Bridge graba ambos canales y registra las transcripciones.

4. Acción de la IA

Si la IA resuelve la consulta, finaliza la llamada. Si requiere intervención humana, solicita una transferencia a una extensión específica.

5. Registro y cierre

Se almacenan en base de datos: duración, transcripciones, grabaciones (WAV dual) y motivo de finalización. El panel refleja los datos en tiempo real.

Componente Puerto Protocolo
AudioSocket Server 9092 TCP binario
Conexión IA → 443 WSS / TLS
AMI (control PBX) → 5038 TCP
API REST + Panel 8080 HTTP + SSE
Base de datos Local SQLite WAL

Grabación dual: Se generan dos archivos WAV independientes por llamada — uno con el audio del llamante y otro con el audio generado por la IA — facilitando la auditoría y el análisis posterior.

Requisitos de integración

¿Qué necesitamos del módulo de IA del CTN?

Audio Bridge no incluye IA propia. Se conecta al módulo que el CTN proporcione. Para ello necesitamos la siguiente información técnica.

Necesitamos Descripción Ejemplo
Endpoint de conexión URL del servicio de IA donde el Bridge enviará el audio en tiempo real wss://ia.ctn.es/v1/realtime
Formato de audio Códec y frecuencia de muestreo que acepta el módulo de IA G.711 μ-law, 8kHz
Autenticación Método para conectarse al servicio (clave API, token, certificado...) API Key / Bearer Token
Protocolo de eventos Formato de los mensajes que la IA envía para solicitar acciones (transferir, colgar...) JSON vía WebSocket
Parámetros de contexto Qué datos acepta la IA al inicio de cada llamada notaría, llamante, horario

En resumen: Necesitamos un documento de integración (API) del módulo de IA. Con esa especificación, Audio Bridge se adapta para comunicarse con el sistema del CTN. El Bridge está preparado para trabajar con cualquier proveedor de IA.

Transferencia a humano

¿Qué ocurre cuando la IA no puede resolver?

La llamada se devuelve a la centralita de la notaría para que la atienda una persona. El proceso es transparente para el llamante.

1. El llamante habla con la IA

La centralita mantiene la llamada activa en todo momento. El audio se redirige al Bridge, que lo traduce para la IA.

2. La IA detecta que necesita ayuda

La IA responde: «Entiendo, le transfiero directamente con el notario. Un momento por favor.»

3. La IA solicita la transferencia

El módulo de IA envía un evento al Bridge indicando a qué extensión transferir (ej: extensión 201).

4. El Bridge ordena a la centralita

Mediante AMI, el Bridge le dice a Asterisk: «redirige esta llamada a la extensión 201». Asterisk ejecuta la transferencia.

5. Llamante conectado con el notario

La llamada queda conectada con la extensión humana. El Bridge se desconecta, guarda la grabación y registra todo.

La llamada nunca se corta

Asterisk no «suelta» la llamada mientras la IA la atiende — solo redirige el audio. Cuando llega la orden de transferir, simplemente cambia el destino. Para el llamante es una experiencia fluida, como una transferencia normal.

Experiencia del llamante

00:00 — Llama al número de la notaría
00:02 — La IA le saluda y le atiende
00:45 — Pide hablar con el notario
00:46 — «Le paso con el notario, un momento»
00:48 — Está hablando con el notario

Tiempo de transferencia: 1-2 segundos. Transparente para el llamante.

Multi-Notaría

Gestión independiente por notaría

Cada notaría se configura como un tenant independiente con sus propios parámetros

Identificación y enrutamiento

  • DDI como clave de enrutamiento: cada número DDI se asocia a una notaría
  • Identificador único por notaría (N001, N002...)
  • Soporte para múltiples DDIs por notaría
  • Trunk SIP único compartido, enrutamiento por DDI

Configuración personalizable

  • Horarios de atención con zona horaria
  • Extensiones (recepción, notario, oficial)
  • Grupo de salto para distribución de llamadas
  • Activación/desactivación independiente

Ejemplo de configuración por tenant

# Configuración de notaría tenants: - notaria_id: "N001" name: "Notaría García López - Barcelona" ddis: ["934001234", "934001235"] # trunk SIP compartido, routing por DDI enabled: true schedule: timezone: "Europe/Madrid" business_hours: - days: "mon-fri" start: "09:00" end: "14:00" - days: "mon-fri" start: "16:00" end: "19:00" transfers: default: "201" extensions: recepcion: "200" notario: "201" oficial: "202"

Conclusiones

Respuesta a las cuestiones planteadas

Audio Bridge responde a los tres escenarios de implantación solicitados

✓ Complemento a la telefonía existente

Audio Bridge se integra con la centralita actual de cada notaría sin necesidad de modificarla. Se añade como un destino AudioSocket en el dialplan, respetando toda la infraestructura existente.

✓ Pieza adicional centralizada

Puede desplegarse como servicio centralizado en la infraestructura del CTN, recibiendo llamadas de múltiples notarías a través de un único trunk SIP compartido, diferenciando cada notaría por DDI.

✓ Ambos casos simultáneamente

La arquitectura multi-tenant permite operar en ambos modos de forma simultánea. Cada notaría se conecta según su propia infraestructura.
